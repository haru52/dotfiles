# CLAUDE.md

## Repository purpose

haru's personal dotfiles. Targets macOS and Linux (including WSL2). Installation is driven by `make` / `install.sh`, which symlinks config files from `dotfiles/` into `~` and scripts from `bin/` (and `private_bin/`, if present) into `~/bin`.

## Common commands

|                        Task                         |                                Command                                 |
| --------------------------------------------------- | ---------------------------------------------------------------------- |
| Install dotfiles onto the system                    | `make` (alias for `make install` → `./install.sh`)                     |
| Set up dev tooling (npm + Vale styles)              | `make dev-install`                                                     |
| Run all linters                                     | `make lint`                                                            |
| Lint one category                                   | `make lint-npm` / `make lint-text` / `make lint-yaml` / `make lint-sh` |
| Regenerate `.gitignore` from gibo                   | `make update-gi`                                                       |
| Regenerate global gitignores shipped in `dotfiles/` | `make update-gi-global`                                                |
| Conventional-commit helper                          | `npm run commit` (or `npm run cm`)                                     |

`make lint-npm` runs `npm run lint`, which fans out via `run-s --continue-on-error` to `secretlint` (skipped when `CI=true`), `markdownlint`, and `prettier --check`. `make lint-text` runs Vale on the top-level `*.md` files and `.github/`. `make lint-sh` runs ShellCheck with `--enable=all --severity=style` on all top-level `*.sh`, `bin/**/*.sh`, and both husky hooks — match those flags if invoking ShellCheck manually.

There is no test suite; `npm test` intentionally exits 1.

## Architecture

The repo is a two-layer install system:

1. **`dotfiles/`** — files that get symlinked into `$HOME`. `install.sh` iterates `ls -A dotfiles/` and `ln -sfv` each regular file into `~`. Subdirectories under `dotfiles/` are *not* symlinked; they are handled by special cases:
   - `dotfiles/zprezto/*` is copied (not symlinked) into `~/.zprezto/runcoms`.
   - `dotfiles/etc/alias.zsh` is copied into `~/.zprezto/modules/git`.
   - `dotfiles/wsl/{wsl.conf,resolv_template.conf}` is symlinked into `/etc/` only when `get-os.sh` reports `linux` (this branch currently has no WSL-vs-native-Linux guard — see the TODO in `install.sh`).

2. **`bin/`** — shell scripts symlinked into `~/bin` *without the `.sh` suffix* (e.g. `bin/clean-branch.sh` → `~/bin/clean-branch`). `bin/linux/apt-update.sh` is only linked on Linux. If the optional `private_bin` symlink exists (points to a user-private directory), its `*.sh` scripts are installed the same way.

3. **Gitignore generation** — `.gitignore` and the two `dotfiles/.gitignore_global*` files are generated by `gibo`. The `*_custom` files at the repo root hold the hand-written prefix that each generated file starts with; never edit the generated files directly — edit the `*_custom` prefix and re-run `make update-gi` / `make update-gi-global`.

4. **OS detection** — always go through `./get-os.sh`, which prints `mac` or `linux` (and exits 1 on anything else). Do not duplicate `uname` switches elsewhere.

## Conventions

Pulled from `CONTRIBUTING.md` — apply to any new shell code:

- File names: `kebab-case.sh`; variables: `snake_case`.
- Comments/identifiers in English.
- Max line length: 100 half-width chars; 2-space indent.
- Must pass `shellcheck --enable=all --severity=style` cleanly.
- Default shebang is `#!/bin/sh`; use `#!/usr/bin/env bash` only when bash features are actually required (`install.sh` is a legitimate bash example).

Commits follow **Conventional Commits** (enforced by commitlint via the husky `commit-msg` hook). Pre-commit runs `lint-staged`, which applies `secretlint` + `prettier --check` to everything, `markdownlint` + `make lint-text` to Markdown, `make lint-yaml` to YAML, and `shellcheck` to shell files. Branching follows GitHub flow; PR titles follow the same rule as commits.
