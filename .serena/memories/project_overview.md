# Project Overview

## Purpose
haru's personal dotfiles repository. Targets macOS and Linux (including WSL2).
Installation is driven by `make` / `install.sh`, which symlinks config files from
`dotfiles/` into `~` and scripts from `bin/` (and optional `private_bin/`) into `~/bin`.

## Tech stack
- **Primary language**: Shell (POSIX `sh` preferred; `bash` only when necessary — e.g. `install.sh`).
- **Tooling layer**: Node.js (`^22.20.0`) + npm (`^11.6.1`) for lint orchestration only — no runtime code.
- **External CLIs required for dev**: `gibo` (`^2.2.7`), `vale` (`^2.20.1`), `yamllint` (`^1.27.1`),
  `shellcheck` (`>=0.8.0 <1.0.0`).
- **Dev deps (npm)**: `commitlint` + `commitizen` (Conventional Commits),
  `husky` (git hooks), `lint-staged`, `markdownlint-cli`, `prettier`, `secretlint`,
  `npm-run-all`, `npm-check-updates`.

## Repo layout (top level)
- `install.sh` — bash installer (symlinks `dotfiles/*` → `~`, `bin/*.sh` → `~/bin/<name>`).
- `get-os.sh` — prints `mac` or `linux`; exits 1 otherwise. Canonical OS detection.
- `Makefile` — entry point for `install`, `dev-install`, `lint*`, `update-gi*`.
- `dotfiles/` — files symlinked into `$HOME`. Subdirs `zprezto/`, `etc/`, `wsl/` are special-cased.
- `bin/` — shell scripts symlinked into `~/bin` without `.sh` suffix. `bin/linux/` is Linux-only.
- `private_bin` — optional symlink to a user-private directory; installed like `bin/`.
- `.gitignore`, `dotfiles/.gitignore_global*` — GENERATED by `gibo`; edit the `*_custom` prefix files.
- CI/config: `.github/`, `.husky/`, `.vscode/`, `.serena/`, `.claude/`, various lint dotfiles.

## Key design points
- No test suite; `npm test` intentionally exits 1.
- OS detection must go through `get-os.sh` — don't inline `uname` switches.
- `install.sh` iterates only regular files in `dotfiles/` top level; subdirs are handled by
  explicit special cases in the script.
- There is a known TODO in `install.sh`: the Linux branch assumes WSL2 without checking.
